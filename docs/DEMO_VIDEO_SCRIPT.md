# 狮选 LionPick — Demo 视频分镜脚本（3.5–4 min）

> 课题要求 6.1：「建议提供 3-5 分钟 Demo 演示 + 技术讲解」。
> 本脚本按**评分维度**编排，每个镜头标注它在为哪个维度得分。配 `docs/DEMO_RUNBOOK.md`
> 的精确点击顺序使用。**录之前务必做开演前检查（见下），否则冷启动会出现 15-18s 卡顿。**

## 录制前检查（5 分钟，必做）
1. **预热**：`python tools/warm-demo.py`（把脚本里每条 query 跑热，冷→热 ~16×，首字 15-18s→~1s）。
2. **真机**：iPhone 装最新 .ipa，亮度调高，开勿扰，设置→语言=中文。
3. **后端**：确认 `/ready` 返回 ready；后端 URL 是当前隧道地址（baked 进 Config.swift）。
4. **环境**：安静房间录旁白；屏幕录制用 iOS 自带录屏（设置→控制中心）或 QuickTime 连线投屏。
5. **节奏**：每条 query 发出后**等首字出现再讲下一句旁白**；不要抢在 loading 时说完。

## 拍摄风格
- 竖屏真机录屏为主；技术讲解段可切一张架构图（`CLAUDE.md §2` 的图）。
- 旁白用中文、口语、简短；字幕可选。
- **三个必须放大讲的差异化**：① 来源标签防幻觉 ② 反选/否定 ③ 多模态拍照找货。

## 备份与兜底
- **录两遍**：一遍连贯走完；一遍逐场景分段录（出问题只补那一段）。
- **每条 query 提前在脚本里写死**（见 `DEMO_RUNBOOK.md §2`），避免现场打字错。
- **若某条冷启慢**：剪辑时加速 loading 段，或重新 `warm-demo.py` 再录。
- **后端隧道地址会变**：录制当天先确认 Config.swift 里的 URL 仍可达（`/health` 200）。
- **导出**：1080p 竖屏，控制在 5 分钟内；文件名 `LionPick-demo-vYYYYMMDD.mp4`。
