## 一句话全景（开场或简介可用）
> 狮选不是一个聊天框——它是一个**端到端的导购 Agent**：从模糊需求的多模态检索，
> 到多轮砍价式追问、反选过滤，到加购、拼单、下单、降价提醒、复购提醒的**完整交易
> 闭环**，再到账户体系与中英双语，全部跑在我们自建的 **RAG + 流式后端**上。

## 录制前检查（5 分钟，必做）
1. **预热**：`python tools/warm-demo.py`（把脚本里每条 query 跑热，冷→热 ~16×，首字 15-18s→~1s）。
2. **真机**：iPhone 装最新 `.ipa`（当前 `apps/LionPick-v0.2.0-b7.ipa`），亮度调高，开勿扰，设置→语言=中文。
3. **后端**：确认 `/ready` 返回 ready；后端 URL 是当前隧道地址（baked 进 Config.swift）。
4. **多模态先试**：拍照找货是最易翻车的一段——**先单独跑一遍**上传商品图→出卡，确认链路通。
5. **登录态**：预先注册好一个演示账号（手机号验证码即可），偏好 👍/👎 段需要登录态。
6. **环境**：安静房间录旁白；屏幕录制用 iOS 自带录屏或 QuickTime 连线投屏。
7. **节奏**：每条 query 发出后**等首字出现再讲下一句**；不要抢在 loading 时说完。

## 技术讲解怎么穿插（关键）
- **两种载体，别打断节奏**：① 屏幕下方**字幕条**（术语 + 一行白话）② 旁白**顺口带过**。
  技术讲解永远是「配着当前这个操作」讲，不做单独的讲解段落。
- **三个必须放大**：① 来源标签防幻觉 ② 反选/否定 ③ 多模态。
  但这一版也要让评委明确看到**交易闭环 + 账户 + 工程**的完备性——那是「基础功能 35%」的分。

## 技术讲解速查卡（字幕文案 + Q&A 弹药，术语统一用这套）

| 术语 | 一行白话（可直接做字幕） |
|---|---|
| 混合检索 Hybrid | 稠密向量找「意思相近」，BM25 找「词面命中」，两路结果用 RRF 融合 |
| 稠密向量 | `BAAI/bge-small-zh-v1.5`，把商品和 query 编码到同一语义空间比相似度 |
| BM25 + jieba | 经典词频检索，jieba 先把中文切词；补「型号/品牌词」这类精确匹配 |
| RRF 融合 | Reciprocal Rank Fusion(Cormack 2009)，按排名而非分数融合，不受量纲影响 |
| 交叉编码器精排 | 把 query 和每个候选拼在一起重新打分，排序更准；中文 `bge-reranker-base`、纯英文 `bge-reranker-v2-m3`，按语言路由 |
| 来源标签 | `[目录✓]`=取自商品目录、`[推断?]`=模型推断；防止编造价格/优惠 |
| 跨轮约束状态机 | 把品类/品牌/预算/否定记成结构化状态，逐轮合并，话题切换才清 |
| 反选/否定 | 「不要苹果」会跨轮持续过滤；「不要X的Y」会重排成「Y 不要X」 |
| 多模态 | 上传图→多模态 LLM 抽属性 + OpenCLIP ViT-B/32(512-d) 图像向量检索 |
| 货币归一 | 外币按带日期的参考汇率换算 CNY 展示与筛选，留原币种可审计 |
| 交易闭环 | 加购/下单走 chat.py 意图；拼单/降价/复购各自 SQLite 子系统 per-account |
| JWT 会话 | 登录签发 HS256 签名、会过期的令牌，篡改/过期即拒；密码 PBKDF2-SHA256 |
| 双层缓存 | 检索结果缓存 + LLM 响应缓存，重复 query 冷启 15-18s → 约 1s |
| 自动部署 | 推 main → 2 分钟内 `git fetch→reset--hard→重启→/ready 健康检查→失败回滚` |
| 评测 | 92 例黄金集 + 61 例组合集，CI 每次跑；recall@5 0.947/0.832、反选 1.000 |

> 补充口径：`recall@5` 是**分数召回**（命中∩top5 / 期望数）；61 例是对比/多轮重场景，
> 二元「至少出 1 个相关」实测 **0.983**——用户几乎总能看到对的商品。145 件商品目录。
> LLM 为多 provider 可切（demo 用 TokenRouter Claude Haiku 级取 ~2s 首字，可切 Doubao/OpenAI）。

## 旁白脚本（连续版，方便一次性配音）
> 狮选，一个基于 RAG 的多模态电商导购 Agent，原则是懂货、绝不编货，而它能做的远不止聊天。/
> 最基础的模糊需求，它识别意图、从商品库检索、给出理由——底层是稠密向量加 BM25 的混合检索、RRF 融合、再交叉编码器精排。/
> 注意每条事实的来源标签：绿色来自目录、琥珀是推断，库里没有的它一个都不编，这是我们防幻觉的核心。/
> 多轮追问逐步收敛：记住跑鞋，叠加轻量和预算，靠的是跨轮约束状态机。/
> 反选：不要苹果，结果里没有 Apple；再加不要 Sony，约束跨轮叠加，反选准确率一点零。/
> 多模态：拍张实物图，视觉模型识别属性、CLIP 图像向量检索相似款，也支持语音问答。/
> 对比决策：自动抽维度排成对比表；进口商品外币自动换成人民币。/
> 然后是完整的交易闭环——自然语言加购、删第二个、改数量，一句下单走完确认；还有拼单、降价提醒、复购提醒。/
> 完整账户体系，登录签发签名会过期的 JWT 会话；你的赞踩偏好实时影响排序。/
> 界面和助手回复一键中英切换、运行时生效。/
> 工程上流式加混合检索加精排加双层缓存，冷启十几秒降到约一秒，推一次代码两分钟自动部署上云、失败回滚，检索效果全程 CI 自动评测，recall@5 零点九四七、反选一点零。/
> 狮选——懂货、不编货，从检索到下单一气呵成的多模态导购 Agent，谢谢。

### 分 beat 脚本（带画面提示）

**Beat 0 — 总览（0:00–0:08）**
> 画面：架构图全景 — iPhone →（SSE）FastAPI → Chroma + BM25 + 精排 → 多模态 LLM
>
> 「先讲讲狮选背后的技术。整个系统分三层：iPhone 原生客户端、云端 RAG 检索后端、和多模态大模型。」

**Beat 1 — RAG 怎么设计的（0:08–0:48）** ⭐ 重点
> 画面：RAG 管线图，随旁白逐步高亮 稠密向量 / BM25 → RRF → 精排 → 结构化过滤 → 来源标签
