Plumbing is wired (`LLM_PROVIDER=tokenrouter`, key in `server/.env`), but the key needs to be activated in the [TokenRouter console](https://www.tokenrouter.com/console/token) before it works.

### 3.3 — A100 CLIP indexing without touching the driver (next 1-2 hrs)

**Honest answer to "will nvidia-smi interrupt cuda-fuzzing?"**:

- *Running* `nvidia-smi` itself: no, read-only.
- *Fixing* the driver/library mismatch: yes — it requires reloading the nvidia kernel module, which kicks every running CUDA process. cuda-fuzzing right now has no live processes (last log activity 05:08), but if cron schedules it, it could be interrupted.
- **Recommended path: don't touch the driver.** Instead:
  1. In our venv on `uc`: `pip install torch==2.4.1+cu124 --index-url https://download.pytorch.org/whl/cu124` — matches the driver-580 kernel module.
  2. If GPU torch still misbehaves, fall back to `pip install torch --index-url https://download.pytorch.org/whl/cpu`. CLIP ViT-B/32 on 100 product images takes ~20 sec on CPU. Acceptable for a one-shot index build.
- That way `cuda-fuzzing` is never at risk. No coordination needed.

Then `python rag/ingest/embed_image.py` indexes product images into Chroma's `products_image` collection.
